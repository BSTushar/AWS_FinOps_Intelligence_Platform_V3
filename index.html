1
Upload your Source of Truth
This is your master retired/decommissioned list — the file that says "these AWS resources should no longer exist." Drag and drop it into the left zone, or click to browse. Supports .xlsx, .xls, .csv up to 50MB.
Tip: If your file has multiple sheets, a dropdown will appear — pick the right one
2
Upload your Monthly Data
This is this month's active inventory — your March EC2 list, current resource export, or monthly AWS report. Drop it in the right zone. The tool will show you a 3-row preview so you can verify it loaded correctly.
Tip: The Data Quality score (0–100) tells you how clean your file is before you run anything
3
Check the Primary Key Detection
The tool automatically scans all columns and picks the best primary key (e.g. Application_Code, Resource_ID, Instance_ID). You'll see a confidence badge — Green = high confidence, Yellow = check it, Red = switch to Manual and pick the column yourself.
Tip: Use the Auto / Manual toggle anytime if the auto-detection picked the wrong column
4
Choose what you're looking for
Select your objective from the dropdown: Decommissioned (find retired resources still appearing in monthly), New Resources (resources not in your master list), Missing (retired resources already gone from monthly), or Full Reconciliation (everything at once).
5
Click Run Reconciliation
The 5-tier matching engine runs in seconds — even on 10,000+ row files. It handles exact matches, typos, case differences, abbreviations, and reordered words automatically. Blank rows, N/A values, and Excel errors (#REF!) are silently filtered out.
Tip: A warning toast will tell you if any rows were skipped due to blank or invalid key values
6
Review your 6 result tabs
Results are split into clear categories. Use the search box to filter, click column headers to sort, and paginate through large result sets. For fuzzy matches in the Review tab, click Accept or Reject on each row.
7
Export your report
Click Export CSV for a single tab, or Full Report XLSX for a complete multi-sheet workbook with all tabs and a Summary sheet. The export includes every column from your original file — not just the key column.
Tip: The Full Report XLSX is ready to email to leadership as-is
What each result tab means
CONFIRMED
Definite matches. These resources appear in your monthly data AND in the retired list. Exact or normalized match. These need to be actioned.
REVIEW
Possible matches. High similarity (typo, abbreviation, case difference) but not 100% certain. Click Accept to confirm retired, or Reject to mark as clean.
NOT IN SOURCE
New resources. Appear in monthly data but have no match in your source of truth. Could be newly provisioned or misnamed.
CLEAN
Active resources. Not in the retired list. No action needed. These are your healthy, active AWS resources.
MISSING
Already gone. These exist in your retired master list but were NOT found in this month's data. They've already been removed — informational only.
DUPLICATES
Duplicate keys. Same primary key appears more than once in the same file. Could cause false matches. Review and clean your source data.

1
Upload your data file
Drop any AWS export — Cost Explorer CSV, CUR file, custom Excel inventory sheet. This mode works on a single file. You're not comparing two files here — you're asking questions about one.
2
Add filter conditions
Click + Add Condition to build your query row by row. Each condition has three parts: Column (which field to check), Operator (how to check it), and Value (what to look for). You can stack as many conditions as you need — all of them must match (AND logic).
Tip: Example: Column = "Monthly_Cost" → Operator = "Greater than" → Value = "50"
3
Optional: Group By and Sort By
Use Group By to summarise results by a category (e.g. Region, Service Type). Use Sort By to order results — the tool auto-detects numeric columns and sorts them numerically so "$1,200" sorts above "$200" correctly.
4
Read your summary + results
The results section shows summary cards at the top (total matching rows, total cost, average cost) followed by the full matching data table. Every column from your original file is shown.
Tip: Cost columns are automatically detected — $1,234.56, 2.5K, (150.00) all parsed correctly
Available filter operators
Contains
Cell includes your value anywhere. Use for partial name matches — "EC2" matches "EC2-PROD-001"
Equals
Exact match (case-insensitive). Use for specific regions, statuses — "us-east-1"
Greater / Less than
Numeric comparison. Strips $, commas automatically. Use for cost thresholds — "> 50"
Not contains
Exclude rows. Filter out dev environments — "Not contains dev"
Is empty
Find rows with a missing value. Great for finding untagged resources — "Tag is empty"
Is not empty
Find rows that have a value. Use to confirm tagged or configured resources

What each tab shows

DROPS

🆕 NEW

REMOVED
Gone this month. Resources that were in last month but not this month — decommissioned, deleted, or merged.
ALL
Complete picture. Every resource with its before/after cost, delta in $ and %, and change type. Best for the full export.

All 12 available templates
Idle Resources
Cost > $0 but zero or empty utilization. These are running but doing nothing — prime candidates for decommissioning.
Top 10 by Cost
Your 10 highest-spending resources, sorted descending. Always the first thing leadership wants to see.
EC2 / S3 / RDS Only
Filter by AWS service type. Shows only the selected service's resources across your entire file.
Cost > $50 / $100
Quick spend threshold filter. Sorted by cost descending so biggest spenders are at the top.
Untagged Resources
Resources with empty or missing tag columns. Tagging compliance is critical for cost allocation and chargeback.
Zombie EC2
EC2 instances that are stopped or terminated but still incurring costs (EBS volumes, Elastic IPs). Should be cleaned up immediately.
Spend by Region
Groups all resources by region and sums the total cost per region. Useful for multi-region cost governance.
Spend by Service
Groups by AWS service type (EC2, S3, RDS, Lambda etc.) and totals spend per service. Great for executive summaries.
Custom Budget Alert
Enter any budget limit — finds all resources exceeding it. Use this for team-level or project-level budget enforcement.

Data Privacy & Safety
100% Local Processing. All data is processed entirely within your browser. No files, rows, or values are transmitted to any external server, API, or third-party service.
Zero Data Retention. No data is stored, cached, or persisted beyond your active browser session. Closing or refreshing the tab permanently clears all uploaded data.
Air-Gap Compatible. This tool operates fully offline with no internet connection required. It is safe for use in restricted or air-gapped network zones.
No Authentication Required. The tool does not collect, transmit, or store usernames, passwords, employee IDs, or any identity data.
Export Responsibility. Exported CSV/XLSX files contain your original data. Handle all exports in accordance with your organisation's data classification policies.
